### 2. ATTACK VECTORS

**Vector 1: Client-Side Code Manipulation**
- **Entry Point**: js/repos.js, js/orgs.js
- **Technique**: Browser developer tools, proxy interception
- **Impact**: Data theft, session hijacking, privilege escalation
- **Likelihood**: HIGH
- **Mitigation**: Code obfuscation, integrity checks, server-side validation

**Vector 2: Supply Chain Compromise**
- **Entry Point**: package.json dependencies
- **Technique**: Malicious package injection, typosquatting
- **Impact**: Remote code execution, data exfiltration
- **Likelihood**: MEDIUM
- **Mitigation**: Dependency pinning, vulnerability scanning, SRI

**Vector 3: Server Misconfiguration**
- **Entry Point**: .htaccess, server configuration
- **Technique**: Directory traversal, information disclosure
- **Impact**: Source code exposure, configuration leak
- **Likelihood**: MEDIUM
- **Mitigation**: Security headers, access restrictions, regular audits

**Vector 4: Cross-Site Scripting (XSS)**
- **Entry Point**: User input fields, API responses
- **Technique**: Reflected/Stored XSS payloads
- **Impact**: Session theft, defacement, malware distribution
- **Likelihood**: HIGH
- **Mitigation**: Input sanitization, CSP, output encoding

**Vector 5: API Abuse**
- **Entry Point**: GitHub/IBM API integrations
- **Technique**: Credential theft, rate limit bypass
- **Impact**: Data breach, service disruption, financial loss
- **Likelihood**: MEDIUM
- **Mitigation**: API key rotation, rate limiting, monitoring
