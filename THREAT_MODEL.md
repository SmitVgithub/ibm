### 2. ATTACK VECTORS

**Vector 1: Cross-Site Scripting (XSS)**
- **Entry Point**: js/repos.js, js/orgs.js
- **Method**: Inject malicious scripts through repository/organization names
- **Impact**: Session hijacking, data theft, malware distribution
- **Likelihood**: HIGH
- **Mitigation**: Input sanitization, CSP headers, output encoding

**Vector 2: Information Disclosure**
- **Entry Point**: Console.log statements
- **Method**: Access browser developer tools on user machines
- **Impact**: Exposure of API responses, tokens, internal data
- **Likelihood**: MEDIUM
- **Mitigation**: Remove console statements, implement proper logging

**Vector 3: Supply Chain Attack**
- **Entry Point**: package.json dependencies
- **Method**: Compromise npm packages, dependency confusion
- **Impact**: Code execution, backdoor installation
- **Likelihood**: MEDIUM
- **Mitigation**: Lock dependencies, use npm audit, implement SRI

**Vector 4: Server Misconfiguration**
- **Entry Point**: .htaccess file
- **Method**: Exploit permissive configurations
- **Impact**: Directory traversal, unauthorized access
- **Likelihood**: MEDIUM
- **Mitigation**: Harden Apache configuration, disable directory listing

**Vector 5: Man-in-the-Middle (MITM)**
- **Entry Point**: Network communications
- **Method**: Intercept unencrypted traffic
- **Impact**: Data interception, credential theft
- **Likelihood**: MEDIUM (if HTTPS not enforced)
- **Mitigation**: Enforce HTTPS, implement HSTS

### 5. ATTACK SCENARIOS

**Scenario A: XSS Data Exfiltration**
```
1. Attacker creates repository with malicious name containing JavaScript
2. Application renders repository name without sanitization
3. Malicious script executes in victim's browser
4. Script exfiltrates session tokens to attacker's server
5. Attacker gains unauthorized access to victim's account
```

**Scenario B: Supply Chain Compromise**
```
1. Attacker identifies unpinned dependency in package.json
2. Attacker publishes malicious version of dependency
3. Next npm install pulls compromised package
4. Malicious code executes during build or runtime
5. Attacker gains code execution on build server or client browsers
```
