### 2. Attack Vectors and Vulnerable Components

**Vector 1: Client-Side Attacks**
- **Target**: js/repos.js, js/orgs.js
- **Method**: XSS injection through repository/org names
- **Impact**: Session hijacking, data theft, malware distribution
- **Mitigation**: Output encoding, CSP implementation, input validation

**Vector 2: API Exploitation**
- **Target**: Backend API endpoints (inferred from JS files)
- **Method**: Parameter tampering, injection attacks, IDOR
- **Impact**: Unauthorized data access, data manipulation
- **Mitigation**: Server-side validation, parameterized queries, authorization checks

**Vector 3: Supply Chain Attack**
- **Target**: package.json dependencies, Travis CI pipeline
- **Method**: Dependency confusion, malicious package injection
- **Impact**: Code execution, backdoor installation
- **Mitigation**: Dependency pinning, integrity verification, private registry

**Vector 4: Configuration Exploitation**
- **Target**: .htaccess, server configuration
- **Method**: Header injection, path traversal
- **Impact**: Information disclosure, access bypass
- **Mitigation**: Secure configuration review, principle of least privilege

### 4. Attack Trees

**Goal: Steal API Credentials**
```
├── Inspect client-side JavaScript
│   ├── View source code in browser
│   └── Download and analyze JS files
├── Intercept network traffic
│   ├── MITM attack (if no HTTPS)
│   └── Proxy API requests
└── Exploit console logging
    └── Access browser developer tools
```

**Goal: Access Unauthorized Data**
```
├── Exploit IDOR vulnerabilities
│   ├── Enumerate repository IDs
│   └── Modify organization parameters
├── Bypass authentication
│   ├── Session fixation
│   └── Token theft via XSS
└── SQL/NoSQL injection
    └── Manipulate query parameters
```
