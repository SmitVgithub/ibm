### 2. ATTACK VECTORS

**A. Client-Side Attacks**

*Vector 1: Cross-Site Scripting (XSS)*
- **Entry Point**: js/repos.js, js/orgs.js
- **Method**: Inject malicious scripts via repository/org names
- **Impact**: Session hijacking, credential theft, defacement
- **Likelihood**: HIGH (no CSP detected)
- **Mitigation**: Implement CSP, sanitize all outputs

*Vector 2: DOM-Based Attacks*
- **Entry Point**: JavaScript files manipulating DOM
- **Method**: Manipulate URL parameters or local storage
- **Impact**: Data exfiltration, phishing
- **Likelihood**: MEDIUM
- **Mitigation**: Validate all client-side inputs

**B. Server-Side Attacks**

*Vector 3: Directory Traversal*
- **Entry Point**: .htaccess misconfiguration
- **Method**: ../../../etc/passwd style attacks
- **Impact**: Source code disclosure, configuration exposure
- **Likelihood**: MEDIUM
- **Mitigation**: Strict .htaccess rules, disable directory listing

*Vector 4: Information Disclosure*
- **Entry Point**: Console.log statements, error messages
- **Method**: Trigger errors, inspect browser console
- **Impact**: API endpoints, data structures, credentials exposed
- **Likelihood**: HIGH
- **Mitigation**: Remove console statements, implement error handling

**C. Supply Chain Attacks**

*Vector 5: Dependency Confusion*
- **Entry Point**: package.json
- **Method**: Publish malicious package with similar name
- **Impact**: Code execution, data theft
- **Likelihood**: LOW-MEDIUM
- **Mitigation**: Lock dependencies, use private registry
